# Brand Identity Exploration Pipeline

Use parallel sandboxes to explore multiple brand identity directions simultaneously, each with complete visual systems generated by Firefly and designed with Bencium skills.

## Scenario

A fintech startup needs a complete brand identity. Instead of presenting one direction at a time, generate 5 complete brand systems in parallel for stakeholder review.

## The Challenge

Traditional branding process:
1. Designer creates mood board → 1 week
2. Stakeholder feedback → 2 days
3. Refine and iterate → 1 week
4. Create final assets → 1 week

**Total: 3-4 weeks for a single direction**

With parallel sandboxes:
1. Generate 5 complete directions → 2 hours
2. Stakeholder votes on favorite → 1 day
3. Refine winning direction → 2 days

**Total: 3 days to refined direction**

## Implementation

### Step 1: Launch Brand Exploration Forks

```bash
#!/bin/bash
# explore_brands.sh

cd apps/sandbox_workflows

for direction in trust growth future human premium; do
    uv run obox https://github.com/your-org/brand-template.git \
        --branch "brand-${direction}" \
        --model opus \
        --prompt "$(cat <<EOF
Create a complete brand identity system for direction: ${direction}

Use the brand configuration from brand_directions.py for this direction.

## Phase 1: Generate Visual Assets with Firefly

Generate these images using the visual_prompts for this direction:
1. Hero image (1920x1080, 16:9)
2. Pattern/texture (seamless, 512x512)
3. Icon set concept (show 6 icons in grid)
4. Team/culture photo (1200x800)
5. App mockup background (1080x1920)

For each image:
- Use generate_image with the specified content_class
- Apply style_transfer from hero to other images for consistency
- Save to /assets/generated/

## Phase 2: Design System with Bencium

Use the specified bencium_skill to create:

1. Logo concepts (3 variations):
   - Wordmark
   - Symbol + wordmark
   - Symbol only

2. Typography scale:
   - H1-H6 sizes
   - Body text
   - Captions
   - Display text

3. Color system:
   - Primary/secondary/accent applications
   - Tint/shade variations (100-900)
   - Semantic colors (success, warning, error)

4. Component library:
   - Buttons (primary, secondary, ghost)
   - Cards
   - Form inputs
   - Navigation

## Phase 3: Brand Applications

Create mockups showing the brand applied to:
1. Website homepage
2. Mobile app screens (3)
3. Business card
4. Email template
5. Social media templates (3 sizes)

## Phase 4: Brand Book

Generate a complete brand book PDF with:
- Brand story and values
- Logo usage guidelines
- Color specifications
- Typography guidelines
- Photography style guide
- Do's and don'ts
- Asset downloads

Save everything to /brand-book/
Commit with message: "Brand identity: ${direction} direction"
EOF
)" &
done

wait
echo "All 5 brand directions generated!"
```

### Step 2: Firefly Asset Generation Per Fork

```python
# Inside each sandbox: generate_brand_assets.py
import asyncio
from firefly_sdk import FireflyClient

async def generate_brand_assets(direction: dict):
    """Generate all visual assets for a brand direction."""
    client = FireflyClient()
    assets = {}

    # Generate hero image first (sets the visual tone)
    hero = await client.generate_image(
        prompt=direction["visual_prompts"]["hero"],
        content_class=direction["firefly_style"],
        width=1920,
        height=1080,
        seed=42  # Reproducible for iteration
    )
    assets["hero"] = hero.images[0]

    # Generate pattern with style consistency
    pattern = await client.generate_image(
        prompt=direction["visual_prompts"]["pattern"],
        content_class=direction["firefly_style"],
        width=512,
        height=512
    )
    assets["pattern"] = pattern.images[0]

    # Generate remaining assets with style transfer from hero
    for key in ["icons", "team", "app_mockup"]:
        base = await client.generate_image(
            prompt=direction["visual_prompts"][key],
            content_class=direction["firefly_style"],
            width=1200 if key == "team" else 1080,
            height=800 if key == "team" else 1920
        )

        # Apply subtle style transfer for brand consistency
        styled = await client.apply_style_transfer(
            prompt=direction["visual_prompts"][key],
            style_image_url=assets["hero"].url,
            style_strength=0.3  # Subtle consistency
        )
        assets[key] = styled.images[0]

    # Generate color variations of pattern
    for variant in ["light", "dark", "accent"]:
        variation = await client.generate_similar_images(
            reference_image_url=assets["pattern"].url,
            prompt=f"{direction['visual_prompts']['pattern']}, {variant} variation",
            num_variations=1
        )
        assets[f"pattern_{variant}"] = variation.images[0]

    return assets
```

### Step 3: Brand Book Generation

Each fork creates a complete brand book:

```typescript
// generate_brand_book.ts
interface BrandBook {
    direction: string;
    assets: GeneratedAssets;
    designSystem: DesignSystem;
    applications: BrandApplications;
}

async function generateBrandBook(direction: BrandDirection): Promise<BrandBook> {
    // Use Bencium skill for design decisions
    const designSystem = await createDesignSystem(direction);

    // Generate all brand applications
    const applications = {
        website: await generateWebsiteMockup(direction, designSystem),
        mobileApp: await generateAppScreens(direction, designSystem),
        businessCard: await generateBusinessCard(direction, designSystem),
        emailTemplate: await generateEmailTemplate(direction, designSystem),
        socialMedia: await generateSocialTemplates(direction, designSystem)
    };

    // Compile brand book
    const brandBook = {
        cover: generateCover(direction),
        toc: generateTableOfContents(),
        brandStory: generateBrandStory(direction),
        logoGuidelines: generateLogoGuidelines(direction),
        colorSystem: generateColorSpecs(direction.colors),
        typography: generateTypographyGuide(direction.typography),
        photography: generatePhotographyGuidelines(direction),
        applications: generateApplicationExamples(applications),
        downloads: generateAssetList()
    };

    // Export as PDF
    await exportToPDF(brandBook, `/brand-book/${direction.name}-brand-book.pdf`);

    return brandBook;
}
```
